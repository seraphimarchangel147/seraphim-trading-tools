# Gimel Code Review Protocol

## Review Checklist
- [ ] Bug identification
- [ ] Performance optimization opportunities  
- [ ] Compliance with lessons.md patterns
- [ ] Test coverage verification
- [ ] Documentation updates needed

## Standards
- Follow existing codebase patterns
- Maintain backward compatibility
- Update Skill Graph if methodology changes
- Log all findings to Obsidian

## Process
1. Read target file(s)
2. Check self/lessons.md for relevant patterns
3. Identify issues and improvements
4. Suggest minimal impact patches
5. Report findings to Seraphim
